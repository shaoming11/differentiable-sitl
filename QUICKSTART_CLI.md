# Quick Start: CLI Usage

This guide shows how to use the ArduPilot SITL Parameter Identification CLI.

## Prerequisites

1. Python 3.8+ installed
2. Dependencies installed: `pip install -r requirements.txt`
3. An ArduPilot DataFlash `.bin` log file from a real flight

## Step 1: Prepare Your Flight Log

The log should contain:
- IMU data (accelerometer, gyroscope)
- Motor outputs (RCOUT)
- GPS data (recommended)
- EKF health data (recommended)

**Tip:** Fly with varied maneuvers (not just hover) for best results. Include:
- Roll and pitch maneuvers
- Yaw rotations
- Altitude changes
- Forward flight

## Step 2: Measure Your Vehicle

You'll need to know:
- **Mass** (weigh your vehicle in kg)
- **Frame type** (quad_x, quad_plus, hexa_x, octo_x)

Optional but helpful:
- Motor arm length (distance from center to motor in meters)
- Propeller diameter (in inches)
- Motor KV rating

## Step 3: Run Basic Identification

```bash
python -m cli.sysid \
    --log path/to/your/flight.bin \
    --frame quad_x \
    --mass 1.2 \
    --output sitl_params.parm
```

This will:
1. Parse your log file
2. Preprocess and align data
3. Check data quality
4. Identify parameters
5. Validate results
6. Write SITL parameter file

## Step 4: Check Excitation (Optional but Recommended)

Before running full identification, check if your flight data is good:

```bash
python -m cli.sysid \
    --log path/to/your/flight.bin \
    --frame quad_x \
    --mass 1.2 \
    --output dummy.parm \
    --excitation-check-only
```

Example output:
```

======================================================================

  Overall Quality: GOOD

  Parameter Excitation Scores:
    ✓ kT          : 0.92
    ✓ kQ          : 0.72
    ✓ Ixx         : 0.84
    ✓ Iyy         : 0.79
    ⚠ Izz         : 0.45
    ✓ tau_motor   : 0.68
    ⚠ c_drag      : 0.32

  Suggested Maneuvers:
    • Izz: Yaw spin - sustained 360°/s yaw rate for 3 s
    • c_drag: Fast forward flight (>10 m/s) segment
```

If you see warnings, fly additional maneuvers and try again.

## Step 5: Run with All Options

For best results, provide physical measurements:

```bash
python -m cli.sysid \
    --log path/to/your/flight.bin \
    --frame quad_x \
    --mass 1.2 \
    --mass-std 0.05 \
    --arm-length 0.165 \
    --arm-length-std 0.005 \
    --prop-diameter 5.1 \
    --motor-kv 2300 \
    --output sitl_params.parm \
    --report results.json \
    --verbose
```

## Step 6: Use the SITL Parameters

Load the generated parameters into ArduPilot SITL:

```bash
# Start SITL with your parameters
sim_vehicle.py -v ArduCopter --console --map --custom-location=... \
    -P sitl_params.parm
```

Or manually load in MAVProxy:
```
param load sitl_params.parm
```

## Tips for Best Results

### 1. Fly Diverse Maneuvers

Good excitation requires:
- **Roll doublets**: Quick roll left/right
- **Pitch doublets**: Quick pitch forward/back
- **Yaw spins**: Sustained yaw rotation
- **Altitude changes**: Throttle steps up/down
- **Forward flight**: Fly straight at >10 m/s

### 2. Ensure Good GPS Lock

- Wait for 3D GPS fix before takeoff
- Fly in open area away from buildings/trees
- Check EKF health is good (visible in logs)

### 3. Measure Physical Parameters

More accurate measurements → better identification:
- Weigh your vehicle precisely
- Measure motor arm length with calipers
- Record exact propeller size
- Know your motor KV rating

### 4. Check Validation Metrics

Look at the validation RMSE:
- **Roll/Pitch < 2°**: Excellent
- **Roll/Pitch 2-5°**: Good
- **Roll/Pitch > 5°**: Poor (may need better flight data)

### 5. Use Verbose Mode for Debugging

```bash
python -m cli.sysid ... --verbose
```

Shows detailed progress and intermediate results.

## Troubleshooting

### Error: "No healthy EKF segments found"

**Problem**: Log doesn't have good EKF health data.

**Solution**:
- Ensure GPS had 3D fix during flight
- Check EKF innovations were low
- Fly in open area with clear sky view
- Wait for EKF to initialize before flying

### Warning: "Data quality is POOR"

**Problem**: Flight didn't excite all parameters.

**Solution**:
1. Run with `--excitation-check-only` to see which parameters
2. Fly the suggested maneuvers
3. Try again with new log

### Error: "No IMU data found in log"

**Problem**: Log file is corrupted or not a valid .bin file.

**Solution**:
- Verify file is from ArduPilot (not QGroundControl or other)
- Check file is not truncated (complete flight)
- Try parsing with MAVExplorer to verify integrity

### Results Look Wrong

**Checklist**:
- Did you weigh the vehicle accurately?
- Is the frame type correct (quad_x vs quad_plus)?
- Was the flight at least 2-3 minutes with varied maneuvers?
- Check validation RMSE - should be < 2° for attitude

## Advanced Usage

### Multiple Logs

Process multiple logs and average results:

```bash
for log in flight*.bin; do
    python -m cli.sysid \
        --log "$log" \
        --frame quad_x \
        --mass 1.2 \
        --output "params_${log%.bin}.parm" \
        --report "results_${log%.bin}.json"
done
```

### Batch Processing

Create a script:

```bash
#!/bin/bash
FRAME="quad_x"
MASS=1.2
ARM_LENGTH=0.165
PROP_DIA=5.1
MOTOR_KV=2300

for log in logs/*.bin; do
    base=$(basename "$log" .bin)
    python -m cli.sysid \
        --log "$log" \
        --frame "$FRAME" \
        --mass "$MASS" \
        --arm-length "$ARM_LENGTH" \
        --prop-diameter "$PROP_DIA" \
        --motor-kv "$MOTOR_KV" \
        --output "output/${base}_params.parm" \
        --report "output/${base}_report.json" \
        --verbose
done
```
