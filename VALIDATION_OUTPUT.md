# Phase 7: Validation & Output Generation

This phase implements model validation and output generation for the ArduPilot SITL parameter identification pipeline.

## Overview

After parameter identification, we need to:
1. Validate parameters on hold-out data
2. Compute comprehensive validation metrics
3. Generate SITL-compatible parameter files
4. Create detailed reports (JSON and text formats)

## Module Structure

```
src/
├── validation/
│   ├── __init__.py
│   ├── rollout.py        # Hold-out validation and trajectory comparison
│   └── metrics.py        # RMSE computation for attitude, velocity, angular velocity
└── output/
    ├── __init__.py
    ├── parm_writer.py    # SITL parameter file generation
    └── report.py         # JSON and text report generation
```

## Validation Strategy

### Hold-Out Testing

The validation approach uses hold-out testing to ensure parameters generalize:

1. **Split Data**: 80% training, 20% hold-out (chronological split)
2. **Identify**: Optimize parameters on training data only
3. **Validate**: Roll out FDM on hold-out PWM sequence
4. **Compare**: Compare predicted vs. smoothed states from RTS

### Validation Metrics

Comprehensive metrics are computed across all state dimensions:

**Attitude (from quaternions):**
- Roll RMSE (degrees)
- Pitch RMSE (degrees)
- Yaw RMSE (degrees, with wrap-around handling)

**Linear Velocity:**
- vx, vy, vz RMSE (m/s)
- Total velocity magnitude RMSE (m/s)

**Angular Velocity:**
- wx, wy, wz RMSE (rad/s)

**Success Criteria:**
- Attitude RMSE < 2°
- Velocity RMSE < 0.1 m/s

## Output Formats

### 1. SITL Parameter File (.parm)

ArduPilot-compatible parameter file with CSV-like format:

```
# Generated by ardupilot-sysid v0.1.0
# Generation date: 2026-03-29T14:15:33.711898
# Source log: example_flight.bin
# Vehicle: quad_x
# Validation RMSE: roll=1.1° pitch=1.1° yaw=1.1°
#
SIM_ESC_TCONST,0.04
SIM_MASS,1.2
SIM_MOI_X,0.015
SIM_MOI_Y,0.018
SIM_MOI_Z,0.025
SIM_MOT_DRAG,0.0008
SIM_THST_EXPO,0.5
SIM_THST_MAX,38.88
```

**Parameter Mapping:**
- `mass` → `SIM_MASS`
- `inertia[0,1,2]` → `SIM_MOI_X`, `SIM_MOI_Y`, `SIM_MOI_Z`
- `kT` → `SIM_THST_MAX` (converted to total thrust)
- `tau_motor` → `SIM_ESC_TCONST`
- `c_drag` → `SIM_MOT_DRAG`

## API Reference

### Validation Functions

#### `hold_out_validation(params, state_trajectory_holdout, pwm_sequence_holdout, dt)`

Validate identified parameters on hold-out data.

**Arguments:**
- `params`: Dict of identified FDM parameters
- `state_trajectory_holdout`: (T, 10) smoothed states from RTS
- `pwm_sequence_holdout`: (T-1, N) PWM commands
- `dt`: Timestep (seconds)

**Returns:**
Dict with keys:
- `predicted`: (T, 10) predicted states
- `actual`: (T, 10) actual states
- `residuals`: (T, 10) prediction errors
- `timestamps`: (T,) time vector

**Example:**
```python
result = hold_out_validation(
    params_optimal,
    states_holdout,
    pwm_holdout,
    dt=0.01
)
```

#### `split_train_test(segments, test_ratio=0.2)`

Split segments into training and test sets.

**Arguments:**
- `segments`: List of (start_time, end_time) tuples
- `test_ratio`: Fraction for testing (default 0.2)

**Returns:**
- `(train_segments, test_segments)`

#### `compare_trajectories(predicted, actual, state_names=None)`

Compute comparison metrics between trajectories.

**Returns:**
Dict of metrics per state dimension:
- `rmse`: Root mean squared error
- `mae`: Mean absolute error
- `correlation`: Correlation coefficient

#### `summarize_validation_metrics(validation_result)`

Compute comprehensive validation metrics.

**Returns:**
Dict with keys:
- `attitude`: Roll, pitch, yaw RMSE
- `velocity`: Velocity RMSE
- `angular_velocity`: Angular velocity RMSE
- `duration_s`: Validation duration
- `n_samples`: Number of samples

### Metrics Functions

#### `quaternion_to_euler(q)`

Convert quaternion to Euler angles (roll, pitch, yaw).

**Arguments:**
- `q`: (4,) or (N, 4) quaternion [qw, qx, qy, qz]

**Returns:**
- (3,) or (N, 3) Euler angles [roll, pitch, yaw] in radians

#### `compute_attitude_rmse(predicted, actual)`

Compute RMSE for attitude from quaternions.

**Returns:**
Dict with keys `roll_deg`, `pitch_deg`, `yaw_deg`

#### `compute_velocity_rmse(predicted, actual)`

Compute RMSE for linear velocities.

**Returns:**
Dict with keys `vx`, `vy`, `vz`, `v_total`

#### `compute_angular_velocity_rmse(predicted, actual)`

Compute RMSE for angular velocities.

**Returns:**
Dict with keys `wx`, `wy`, `wz`

### Output Functions

#### `convert_to_sitl_params(params)`

Convert identified FDM parameters to ArduPilot SITL parameter names.

**Arguments:**
- `params`: Dict of identified parameters

**Returns:**
Dict mapping SITL parameter name → value

**Example:**
```python
sitl_params = convert_to_sitl_params({
    'mass': 1.2,
    'inertia': np.array([0.015, 0.018, 0.025]),
    'kT': 1.2e-5,
    'pwm_to_omega_poly': np.array([0.0, 900.0])
})
# Returns: {'SIM_MASS': 1.2, 'SIM_MOI_X': 0.015, ...}
```

#### `write_parm_file(output_path, sitl_params, metadata=None)`

Write ArduPilot SITL parameter file.

**Arguments:**
- `output_path`: Path to output .parm file
- `sitl_params`: Dict of SITL parameters
- `metadata`: Optional dict with `log_file`, `frame_type`, `validation_rmse`

#### `generate_json_report(params_identified, confidence_intervals, excitation_scores, validation_metrics, metadata, output_path)`

Generate comprehensive JSON report.

**Arguments:**
- `params_identified`: Optimal parameter values
- `confidence_intervals`: 95% CIs per parameter
- `excitation_scores`: Excitation analysis results
- `validation_metrics`: Hold-out validation metrics
- `metadata`: Log info, preprocessing stats
- `output_path`: Path to output JSON file

#### `generate_text_report(params_identified, confidence_intervals, validation_metrics, output_path=None)`

Generate human-readable text report.

**Arguments:**
- `params_identified`: Optimal parameter values
- `confidence_intervals`: 95% CIs per parameter
- `validation_metrics`: Hold-out validation metrics
- `output_path`: Optional path to output file

**Returns:**
Formatted string (and optionally writes to file)

## Usage Example

See `examples/validation_demo.py` for a complete demonstration.

```python
from src.validation.rollout import hold_out_validation, split_train_test
from src.validation.metrics import summarize_validation_metrics
from src.output.parm_writer import convert_to_sitl_params, write_parm_file
from src.output.report import generate_json_report, generate_text_report

# 1. Split data
train_segments, test_segments = split_train_test(segments, test_ratio=0.2)

# 2. Validate on hold-out data
validation_result = hold_out_validation(
    params_optimal,
    states_holdout,
    pwm_holdout,
    dt
)

# 3. Compute metrics

# 4. Convert to SITL format
sitl_params = convert_to_sitl_params(params_optimal)

# 5. Write outputs
write_parm_file('output.parm', sitl_params, metadata)
generate_json_report(params, cis, excitation, validation, metadata, 'report.json')
generate_text_report(params, cis, validation, 'report.txt')
```

## Implementation Notes

### Quaternion Handling

- Quaternions are normalized before Euler conversion
- Gimbal lock is handled with `np.clip` in pitch calculation
- Yaw wraparound is properly handled with `arctan2(sin, cos)`

### Parameter Conversion

- `SIM_THST_MAX` is computed from `kT` and max RPM for 4 motors (quadcopter)
- For other frame types, adjust `n_motors` in `convert_to_sitl_params`
- `SIM_THST_EXPO` defaults to 0.5 (linear), could be estimated from polynomial fit

### Validation Best Practices

1. **No Data Leakage**: Ensure hold-out data is strictly separate
2. **Chronological Split**: Use time-based splitting, not random
3. **Representative Data**: Hold-out should cover similar maneuvers
4. **Multiple Segments**: Use multiple hold-out segments for robustness

### Report Warnings

Automatic warnings are generated for:
- Poorly excited parameters (excitation score < threshold)
- Large confidence intervals (CI width > threshold)
- Poor validation metrics (RMSE > threshold)
